# Quanto Falta? (What's Missing?)

An iOS app to track progress on the **Panini FIFA World Cup 2026** sticker album (994 stickers, Brazilian edition),
built with SwiftUI + SwiftData. Mirrors the architecture of the sibling `PokeCompanion` app.
Fully localized in **English and Brazilian Portuguese** (`Resources/Localizable.xcstrings`;
display name localizes via `InfoPlist.xcstrings`: "Quanto Falta?" / "What's Missing?").

## Features
- **Album tab** — all 994 stickers grouped by section (Opening → FIFA Museum → 48 teams → Coca-Cola),
  with overall and per-section progress. Tap to mark owned; expand for a copies stepper and name/note editing.
  Filter by **status** (All / Have / Missing), **type** (Brilliant / Players), and **team**; search by name or number.
  **Sort** by Album order (FIFA group draw, A–L) or Alphabetical, ascending/descending.
- **Pages view** (book icon on the Album tab) — the album as swipeable printed pages matching the physical
  layout: team spreads with the "We are…" header, wide team-photo slot, and group box exactly where they
  sit in the real album. Tap a slot to collect; long-press for copies. Jump menu to any section.
- **Duplicates tab** — every sticker you own more than once, with steppers and a running "spare" total. Your trade list.
- **Digital album** — a Physical/Digital switcher on the Album and Duplicates tabs tracks the
  **FIFA Panini Collection** app (798 stickers: intro foils, 48 teams × emblem `0` + players `1-11`,
  #AllTheFeels Coca-Cola set, host city posters, Update Edition, fan/McDonald's/Trophy Tour sets) as a
  separate collection — for planning swap requests. All 528 player + 96 update names are filled from
  the published checklist. Progress is stored per album (digital rows use a `d:` key prefix in the
  same SwiftData table; no migration).
- **Welcome tour & tips** — a first-launch walkthrough (replayable via Settings → Help) plus TipKit
  popovers on the filter, view toggle, album switcher, page swiping, and duplicates controls.
- **Share/Export** — on each tab, share the **Missing** (Album) or **Duplicates** list as paste-ready **text**
  or a **CSV** file. Same columns for both; duplicates carry `×N` spare counts.
- **Settings** — progress stats, include/exclude the Coca-Cola set, welcome-tour replay,
  in-app privacy policy, reset, about.

Storage is **local-only** (SwiftData on-device).

## Catalog data
The sticker structure is exact (codes, sections, foils, 48 teams, flags):
- `00` + `FWC1`–`FWC8` opening foils, `FWC9`–`FWC19` FIFA Museum foils
- 48 teams × 20 (`MEX1`…`MEX20`): pos 1 = team logo (foil), pos 13 = team photo, the rest players
- `CC1`–`CC14` Coca-Cola promo set (Brazilian under-the-label edition; the US set differs)

All **864 player names** are filled from public checklists (`DataPipeline/player_names.json`, compiled from
checklistinsider.com + diamondcardsonline.com and cross-verified). They're best-effort and **editable in-app**
(tap a sticker → Edit) for roster changes or checklist typos. Teams are stored in the album's group-draw order.

Regenerate the bundled catalogs after editing a generator:
```
python3 DataPipeline/generate_stickers.py           # physical album (994)
python3 DataPipeline/generate_digital_stickers.py   # FIFA Panini Collection app (796)
```
The digital structure and names come from the checklist published by
cartophilic-info-exch.blogspot.com (May 2026), reconciled with the user's app for regional sets
(#AllTheFeels = 14, McDonald's = 8). Fan-sticker names are unpublished and stay null.
The app icon is generated by `python3 DataPipeline/make_icon.py` (requires Pillow).

## Build & run
```
xcodegen generate
xcodebuild -project AlbumTracker.xcodeproj -scheme AlbumTracker \
  -destination 'platform=iOS Simulator,name=iPhone 17' build
xcodebuild ... test          # 11 unit tests (catalogs, pages, scoping, exporter)
```
Launch with `-seedSampleData` (Debug) to populate example progress for screenshots.
